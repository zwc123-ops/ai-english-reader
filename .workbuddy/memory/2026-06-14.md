# 2026-06-14 工作日志

## 完成四项优化

为 AI 英语精读阅读器项目实施了四项功能优化，代码改动集中在 `app.js`、`styles.css`、`index.html` 三个文件。

### 1. TTS 朗读功能 ✅
- 使用 Web Speech API（`speechSynthesis`），零依赖零成本
- 新增 `ttsSpeak()`、`ttsStop()`、`ttsIsSpeaking()` 三个核心函数
- 词卡面板添加 `▶` TTS 朗读按钮（`data-panel-action="tts-word"`）
- 句子分析面板添加 `▶ 朗读` 按钮（`data-panel-action="tts-sentence"`）
- 阅读器顶部栏添加 `▶ 朗读` 按钮，支持朗读选中文本或当前段落

### 2. SM-2 间隔复习算法 ✅
- 替换了原来简单的"认识+1/模糊+0/不认识归零"逻辑
- 新增 `computeSm2()` 函数，实现标准 SM-2 算法
- 三档映射：known→5（完美），uncertain→3（及格），unknown→1（失败）
- 新增字段：`interval`（间隔天数）、`easinessFactor`（难度因子，默认2.5）
- 复习提醒 toast 显示下次复习时间（今天/明天/N天后）

### 3. PDF 文本层支持 ✅
- 用 PDF.js 替换原来的 `<object>` 标签嵌入方式
- `renderPdf()` 改为生成占位容器，`initPdfViewer()` 异步渲染页面+文本层
- 开启 `pdfjsLib.renderTextLayer()`，PDF 内容可选择文字
- 文本层绑定 `mouseup` 和 `dblclick` 事件，复用查词逻辑
- `index.html` 引入 PDF.js 3.11.174 CDN
- 添加 `.pdf-container`、`.pdf-text-layer` 等样式，删除旧 `.pdf-frame`

### 4. 流式 Markdown 渲染 ✅
- 引入 marked.js 12.0.2 CDN
- 新增 `renderMarkdown()` 函数，fallback 到 `escapeHtml()`
- `renderAiResult()` 全面改造：所有文本内容改用 `renderMarkdown()` 渲染
- 词卡：语境含义、常见释义、例句、学习提示 → Markdown
- 句子分析：概览、主干、翻译、学习建议、训练题 → Markdown
- 添加 `.markdown-body` 系列样式（段落、粗体、斜体、代码、列表、引用块）

### TTS 朗读无声 Bug 修复 ✅
- **根因**：Chrome Web Speech Synthesis 三个经典问题
  1. `speechSynthesis.getVoices()` 异步加载，首次调用时 voices 为空，speak 静默失败
  2. 未设置 utterance.voice 属性，浏览器可能选了不发声的默认 voice
  3. Chrome speechSynthesis 在页面空闲后容易"卡死"，后续调用全部无声
- **修复方案**：
  - 新增 `initTts()` + `tryLoadTtsVoices()`，监听 `voiceschanged` 事件预加载语音列表
  - 自动选择最优英语 voice（en-US female > Google US > en-US > en-GB > any en）
  - speak 前先 cancel，延迟 50ms 再调用（Chrome hack）
  - 添加 onerror/onstart/onend 事件处理
  - 监听 visibilitychange，页面隐藏时自动 cancel（防止后台卡死）
